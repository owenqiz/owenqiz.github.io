EDA on Bangladesh Groundwater Arsenic Part 1

February 8, 2019 - 19 minutes
Statistics Spatial

This post is a practice note based on tutorials from tutorial 1, tutorial 2 and tutorial 3 towards my thesis, we first perform EDA on data of original scale, once we have idea about our data, we may further perform kriging.

Data was downloaded from British Geological Survey as “DPHE/BGS National Hydrochemical Survey”, describing various groundwater quality parameters for several thousand wells measured by the British Geological Survey throughout Bangladesh in 1998-1999. The dataset contains the results of the survey of 3534 boreholes from 61 out of 64 districts in Bangladesh. 34 variables are recorded, with 14 descriptive and 20 measured ones, we only focusing on the Arsenic level which has the measuring unit of \(\mu\)g/L.

Note that among all 3534 observations, 1105 observations are cencored data of two type, 111 measurements are labeled < 6 and 944 measurements are labeled < 0.5. The remaining 2429 are precised measurements.

We are going to use R package geoR to perform the EDA, which does NOT have functions to handle the censored data, we start with only the non-censored data. We will perform the EDA on two different scale on three places.

  1. All Bangladesh (2429)
  2. A sub-region has high arsenic levels. (Comilla, 101)
  3. A sub-region does NOT have high arsenic levels. (Kishoreganj, 92)

1. Data Reading and Cleaning

We first read the data from the BGS website.

# Arsenic data download from http://www.bgs.ac.uk/home.html
# http://www.bgs.ac.uk/downloads/start.cfm?id=2214

# read csv file from British Geological Survey website

# rename variable by lowercase
colnames(NationalSurveyData) <- tolower(colnames(NationalSurveyData))

Then, we only select column 4,5,9,10,11,15 which correspond to lat_deg, long_deg, division, district, thana, as. Put it into a new dataframe called as_level.

as_level <- NationalSurveyData[,c(4,5,9,10,11,15)]

Now we select these two types of censored data, and label them as -1, -2 respectively for later manipulations.

censor_6 <- subset(as_level, as_level$as == "< 6")
censor_p5 <- subset(as_level, as_level$as == "< 0.5")

censor_6$as <- rep(-1, 111)
censor_p5$as <- rep(-2, 994)

We then sort the data according to the as level, note that the previous steps will make the censored data on the top rows. After removing the censored data, we can change the factor into numeric values as we want.

# find the number of censored data and sort then remove
num_censor <- nrow(subset(as_level, as_level$as == "< 6")) + nrow(subset(as_level, as_level$as == "< 0.5"))

noncensor <- as_level[order(as_level$as),] 
noncensor <- noncensor[-(1:num_censor),]

# convert observation into number
x <- noncensor$as
x <- as.numeric(levels(x))[x]
noncensor$as <- x

Now we have the dataframe noncensor which contains all precised measured data of observation 2429 and further subsetting into sub-regions.

For computation and exploratory purpose, we use the subset the data with respect to different level of sub-regions by R function split, for starters, we want to choose the sub-regions who have two conditions,

  1. The number of observations are around 100.
  2. One region with more high level arsenic observations, and another region has less high level arsenic observations.

An example of the code to look for second level sub-regions (district) is

# 2nd level district
district <- split(noncensor, noncensor$district)

dist_name <- attributes(district)[[1]][-1]
size <- sapply(district, nrow)[-1]
dist <- cbind(dist_name, size)

According to the criteria and the table, we have two candidates, namely Comilla and Kishoreganj as mentioned above. We subset the dataframes from the big one.

comilla <- district$Comilla
kishoreganj <- district$Kishoreganj

We first do a quick data display with exploratory checks on spatial patterns, trends in the coordinates, and distribution of the data. We are using the default plot for a geodata object in geoR::plot.geodata. The lowess argument adds a smooth line to the upper right scatter plot (data value vs y value) and to the lower left scatter plot (x value vs data value). The upper left plot shows color coded data quartile at the location of the data, fefault colors for the quartiles in the first plot are (from 1st to 4th) blue, green, yellow and red.

4. Variogram

Accordig to the documentation, variog implement two types of estimation on semivariance \(\hat{\gamma}(h)\), we will use the default, i.e. the one computes the classical method of moments estimator. \[\hat{\gamma}(h) = \frac{1}{2N_h}\sum_{i = 1}^{N_h}[Y(x_{i+h}) - Y(x_{i})]^2\]

Similarly, we use ggplot2 to combine different types of variogram for each region, we examine the omnivariogram of bined and smooth version, also combine with 4 directional variogram. Note that, if duplicates are exist, we will NOT able present the 4 directional variogram.

Note that for computation purpose, big regions (here we set samples \(\geq\) 500) will only present the binned omnidirectional variogram.

variogram <- function(data, trend = "cte"){
  # max.dist NOT specified, will use the max of pairwise distance
  # trend have const = "cte", linear = "1st", quadratic = "2nd"
  # in basic_plot we assume it is constant
  
  omni_vario_bin <- variog(data, trend = trend)
  df_omni_vario_bin <- as.data.frame(cbind(semivariance = omni_vario_bin$v, distance = omni_vario_bin$u))
  
  p1 <- ggplot(data = df_omni_vario_bin, aes(x = distance, y = semivariance)) + 
    geom_point() + xlim(0, NA) + ylim(0,NA) + theme_bw()
  
  if(length(data[[2]]) >= 500){
    return(p1)
  }
  else{
  omni_vario_sm <- variog(data, trend = trend, option = "smooth")
  df_omni_vario_sm <- as.data.frame(cbind(semivariance = omni_vario_sm$v, distance = omni_vario_sm$u))
 
  p2 <- ggplot(data = df_omni_vario_sm, aes(x = distance, y = semivariance)) + 
    geom_point(alpha = .75) + xlim(0, NA) + ylim(0,NA) + theme_bw()

  # check if the area contains duplicates
    if (length(summary(data)) == 5){ # exist duplicates
        p <- ggarrange(p1, p2, ncol = 1, nrow = 2)
        return(p)
    } 
  else{ # no duplicate length of summary is 4
    direct4_vario <- variog4(data, trend = trend)
    u <- sapply(direct4_vario, `[[`, 1)[,-5]
    v <- sapply(direct4_vario, `[[`, 2)[,-5]
    lab <- rep(c(0,45,90,135), each = 13)
    df_direct4_vario <- as.data.frame(cbind(semivariance = c(v), distance = c(u), direction = lab))

    p3 <- ggplot(data = df_direct4_vario) + xlim(0, NA) + ylim(0,NA) + theme_bw() +
      geom_line(aes(x = distance, 
                    y = semivariance, 
                    group = direction, 
                    col = as.factor(direction),
                    linetype = as.factor(direction)
      ), size = 1) + theme(legend.title = element_blank())
    
    p <- ggarrange(p1, p2,p3, ncol = 1, nrow = 3)
    return(p)  
    }
  }
}

4. Model estimation

There are 4 methods for estimating the (covariance) model parameters in geoR.

  • interactive fitting: using the function `eyefit.
  • non-linear using the function variofit basically fitting a non-linear model to the empirical variogram.
  • likelihood (ML or REML) fitting: using the function likfit.
  • Bayesian fitting using the function krige.bayes.

I will use variofit(taking variog object as argument) and likfit (use geodata as input). According to the empirical variogram, we will NOT set the nugget effect to be zero, but either given a fixed value or estimate the parameter \(\hat{\tau}^2\). The default covariance model is matern which has the form \[\gamma(h) = \left\{\begin{array}{ll} 0 & ,h = 0\\ \tau^2 + \sigma^2\left(1-\frac{(\phi h)^\nu}{2^{\nu-1}\Gamma(\nu)}K_\nu(\phi h)\right) &, h >0 \end{array}\right.\] where \(\Gamma\) is the gamma function and \(K_\nu\) is the modified Bessel function of the second kind.

In terms of the ggplot2 implementation, a post has discuss how to convert a likfit object or variofit object into dataframe, a working auxiliary function can be found on Github Gist

# convert a variofit model into dataframe
# source: https://gist.github.com/cwickham/4027851
vario2df <- function (x, max.dist, scaled = FALSE, ...) {
  my.l <- list()
  if (missing(max.dist)) {
    my.l$max.dist <- x$max.dist
    if (is.null(my.l$max.dist)) 
      stop("argument max.dist needed for this object")
  }
  else my.l$max.dist <- max.dist
  if (any(x$cov.model == c("matern", "powered.exponential", 
                           "cauchy", "gencauchy", "gneiting.matern"))) 
    my.l$kappa <- x$kappa
  else kappa <- NULL
  if (is.vector(x$cov.pars)) 
    my.l$sill.total <- x$nugget + x$cov.pars[1]
  else my.l$sill.total <- x$nugget + sum(x$cov.pars[, 1])
  my.l$nugget <- x$nugget
  my.l$cov.pars <- x$cov.pars
  my.l$cov.model <- x$cov.model
  if (scaled) {
    if (is.vector(x$cov.model)) 
      my.l$cov.pars[1] <- my.l$cov.pars[1]/my.l$sill.total
    else my.l$cov.pars[, 1] <- my.l$cov.cov.pars[, 1]/my.l$sill.total
    my.l$sill.total <- 1
  }
  gamma.f <- function(x, my.l) {
    if (any(my.l$cov.model == c("linear", "power"))) 
      return(my.l$nugget + my.l$cov.pars[1] * (x^my.l$cov.pars[2]))
    else return(my.l$sill.total - cov.spatial(x, cov.model = my.l$cov.model, 
                                              kappa = my.l$kappa, cov.pars = my.l$cov.pars))
  }
  x_vals <- seq(0, my.l$max.dist, length.out = 100)
  data.frame(x = x_vals, fit = gamma.f(x_vals, my.l = my.l))
}
variofit_plot <- function(data, nug = 10^4){
  
  vg <- variog(data)
  df_vg <- as.data.frame(cbind(semivariance = vg[[2]], distance = vg[[1]]))
  
  # fixed nugget effect
  ml_fn <- likfit(data, ini = c(1,0.5), fix.nugget = TRUE, nugget = nug)
  reml_fn <- likfit(data, ini = c(1,0.5), fix.nugget = TRUE, nugget = nug, method = "RML")
  ols_fn <- variofit(vg,ini = c(1,0.5), fix.nugget = TRUE, nugget = nug, weights="equal")
  wls_fn <- variofit(vg, ini = c(1,0.5), fix.nugget = TRUE, nugget = nug)
  
  # dataframe for 4 different estimations
  lab <- rep(c("ML", "ReML","OLS","WLS"), each = 100)
  
  df1 <- lapply(list(ml_fn, reml_fn, ols_fn, wls_fn), vario2df)
  u <- df1[[1]][,1]
  v <- sapply(df1, `[[`, 2)
  df1_vg_line <- cbind(semivariance = c(v), distance = u)
  df1_vg_line <- data.frame(df1_vg_line, method = lab)

  ymax <- 1.1 * max(df_vg$semivariance)
  p1 <- ggplot(data = df_vg, aes(x = distance, y = semivariance)) + geom_point() + xlim(0, NA) + ylim(0,ymax)
  p1 <- p1 + geom_line(data = df1_vg_line, aes(x = distance, y = semivariance, 
                     group = method, col = method, linetype = method), # colour and linetype for different method 
                     size = 1, na.rm = TRUE) + theme_bw() + ggtitle("Fixed Nugget Effect") +
       theme(plot.title = element_text(hjust = 0.5))
  
  # estimated nugget effect
  ml_n <- likfit(data, ini = c(1,0.5), nug = 10^4)
  reml_n <- likfit(data, ini = c(1,0.5), nug = 10^4, method = "RML")
  ols_n <- variofit(vg, ini = c(1,0.5), nugget= 10^4, weights="equal")
  wls_n <- variofit(vg, ini = c(1,0.5), nugget=10^4)
  
  df2 <- lapply(list(ml_n, reml_n, ols_n, wls_n), vario2df)
  u <- df2[[1]][,1]
  v <- sapply(df2, `[[`, 2)
  df2_vg_line <- cbind(semivariance = c(v), distance = u)
  df2_vg_line <- data.frame(df2_vg_line, method = lab)
  
  p2 <- ggplot(data = df_vg, aes(x = distance, y = semivariance)) + geom_point() + xlim(0, NA) + ylim(0,ymax)
  p2 <- p2 + geom_line(data = df2_vg_line, aes(x = distance, y = semivariance, 
                                              group = method, col = method, linetype = method), # colour and linetype for different method 
                      size = 1, na.rm = TRUE) + theme_bw() + ggtitle("Estimated Nugget Effect") +
    theme(plot.title = element_text(hjust = 0.5))
  p <- ggarrange(p1, p2, nrow = 1, ncol = 2, common.legend = TRUE, legend = "bottom")
  return(p)
}
